# Grafana dashboard psistat-exporter

Dashboard `psistat-exporter-dashboard.json` показывает состояние exporter'ов Linux PSI и динамику pressure по ресурсам `cpu`, `io`, `memory`.

## Переменные

- `datasource` - Prometheus datasource.
- `job`, `env`, `group`, `instance` - иерархия выбора targets. Метки `env` и `group` должны приходить из Prometheus scrape labels.
- `resource` - фильтр ресурсов PSI: `cpu`, `io`, `memory`. Можно выбрать несколько значений или `All`.
- `interval` - пользовательский интервал для будущих/производных панелей.

## Панели

### Exporter Instances

Таблица по выбранным instance. Показывает окружение, группу, job, версию exporter'а, состояние последнего сбора, наличие ошибки, настроенный порог события, интервал события и возраст последнего успешного сбора.

Интерпретация:

- `Collector Success = OK` - последний цикл чтения `/proc/pressure/*` прошёл успешно.
- `Collector Success = Failed` или `Last Error Present = Yes` - exporter не смог обновить данные; проверьте доступ к `/proc/pressure/*` и состояние процесса.
- `Collection Age` должен быть близок к scrape interval. Большое значение означает, что exporter перестал обновлять snapshot или Prometheus давно не видел свежих данных.
- `Version` берётся из метрики `psistat_info{version=...}`.

### Short PSI Some

Показывает короткие окна `1s`, `3s`, `10s` для PSI `some`.
`some` означает, что часть задач ждала ресурс, но система не обязательно была полностью заблокирована.

Высокие короткие пики полезны для поиска кратковременных задержек. Для `cpu` это конкуренция за CPU, для `io` ожидание диска, для `memory` давление памяти.

### Short PSI Full

Показывает короткие окна `1s`, `3s`, `10s` для PSI `full`.
`full` означает, что все non-idle задачи были заблокированы ожиданием ресурса. Это более жёсткий сигнал, чем `some`.

Даже небольшие устойчивые значения `full` обычно требуют внимания, особенно для `memory` и `io`.

### Long PSI Some

Показывает kernel-окна `60s` и `300s` для PSI `some`.
Эта панель сглаживает краткие всплески и помогает отличать разовый пик от устойчивой деградации.

Если `avg300` растёт, pressure уже держится несколько минут.

### Long PSI Full

Показывает kernel-окна `60s` и `300s` для PSI `full`.
Устойчивые ненулевые значения говорят о серьёзной нехватке ресурса.

### Threshold Events In Range

Показывает прирост `psistat_event_total` за выбранный диапазон времени dashboard.
Событие возникает, когда PSI выбранного event interval превысил настроенный threshold.

Используйте панель для ответа на вопрос: были ли превышения порога в текущем окне просмотра.

### Threshold Event Counter

Показывает накопительный счётчик threshold events с момента старта exporter'а.
Рост счётчика означает повторяющиеся превышения порога.

### Last Event Percent

Показывает PSI percent, на котором был зафиксирован последний threshold event.
Чем значение выше порога, тем сильнее был pressure в момент события.

### Last Event Age

Показывает, сколько секунд прошло с последнего threshold event.
Малое значение означает недавнюю проблему; большое значение при стабильном счётчике обычно означает, что новых превышений не было.

### Total Stalled Seconds Since Boot

Показывает накопленное время stall по данным ядра с момента загрузки.
Это counter-like показатель: он растёт со временем и полезен для сравнения ресурсов и instance, но сам по себе не показывает текущую интенсивность проблемы.

### Collection Duration

Показывает длительность внутреннего цикла сбора exporter'а в миллисекундах.
В норме значения должны быть маленькими и стабильными. Рост может означать проблемы с чтением PSI-файлов, нагрузку на host или зависания процесса.
